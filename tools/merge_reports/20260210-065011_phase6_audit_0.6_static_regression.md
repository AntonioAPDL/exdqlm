## Dependency / Compatibility Notes

### `regMod`
- API in source: `regMod(X, m0, C0)`.
- Output contract matches current builder pattern (returns `list(FF, GG, m0, C0)` with class `exdqlm`).
- Source examples use `combineMods`; current branch uses `+.exdqlm`. This is a docs compatibility issue, not core logic.

### Static inference functions
- `exal_static_mcmc(y, X, p0, ...)` and `exal_static_LDVB(y, X, p0, ...)` are standalone static-model fitters.
- They depend on existing package utilities (`A.fn`, `B.fn`, `C.fn`, `L.fn`, `U.fn`) and existing C++ sampling bridges (`sample_truncnorm`, `sample_gig_devroye_vector`) already present on current branch.
- No new `src/` code appears required for minimal functionality.
