# Reproducibility Notes

All launch, smoke, and preparation commands must record:

- `R.version.string`
- `Sys.which("Rscript")`
- `R.home()`
- `.libPaths()`
- repository branch and commit
- run tag
- shared source root
- source file hashes
- full row manifest path
- stage manifest path

R 4.6.0 or newer is required. On Muscat, plain `Rscript` should resolve to:

`/data/jaguir26/local/opt/R/4.6.0/bin/Rscript`

Do not call `/usr/bin/Rscript` for this study.

The exDQLM/DQLM harness consumes the frozen shared source bundle generated by
the Q-DESN validation side. It does not generate, append, or splice sources.

## Provisional Progress Snapshots

Live progress snapshots are reproducible generated artifacts, not source files.
The Q-DESN TT500 provisional exporter reads the frozen atomic launch plan and
the current per-row scalar summaries, then rewrites:

- `tt500_provisional_atomic_progress.csv`
- `tt500_provisional_root_progress.csv`
- `tt500_provisional_manifest.json`
- `README.md`

under the campaign `provisional_progress/` report directory. These outputs are
storage-light and updateable, but intentionally non-authoritative while any
atomic spec remains running or pending. Every provisional row must carry
`is_final = FALSE` and `article_consumable = FALSE`; final Article-Q-DESN
scientific result table inputs require the later shared fit+forecast interface
closeout export. Article-Q-DESN may use the snapshot only for progress/preflight
checks or explicitly labeled operational status displays.

Refresh command for the active TT500 campaign:

```sh
Rscript scripts/export_qdesn_tt500_provisional_progress_table.R \
  --run-tag qdesn-dynamic-fitforecast-v2-mcmc-tt500-20260520-035319__git-d075941 \
  --campaign-id 20260525-191523__git-d075941
```
