\section{Purpose and scope}
This document specifies a rigorous, time-series-safe model selection workflow for Q-DESN, where multiple quantile models (indexed by quantile level $\tau$) share a common DESN specification and are synthesized into a single predictive distribution. The goal is to select DESN hyperparameters efficiently and reproducibly using out-of-sample predictive performance of the \emph{synthesized} distribution (primary) plus calibration of the \emph{individual} quantile models (secondary/constraint or penalty).

\paragraph{Selection target.}
We select the shared DESN specification while holding secondary parameters fixed. In repo terms, the selection target is the \texttt{cfg$desn} block (e.g., \texttt{D, n, n\_tilde, m, alpha, rho, pi\_w, pi\_in, washout}) with readout and VB settings held fixed unless explicitly varied.

\paragraph{Key design principles.}
\begin{enumerate}[label=(\arabic*), itemsep=2pt, topsep=4pt]
\item Strict time-series evaluation (no leakage): train/verification/test split and rolling-origin evaluation on the verification block.
\item Proper scoring rules for distributional forecasts: CRPS for the synthesized predictive distribution.
\item Explicit quantile calibration assessment: coverage/PIT or calibration error per $\tau$.
\item Explicit handling of ESN stochasticity: common random numbers across candidates; average objectives across seeds.
\item Multi-stage, multi-fidelity search: broad screening, then racing/successive halving, then final robust evaluation.
\item Caching and parallelization: reuse reservoir states per (candidate, seed) across quantiles; parallelize across candidates/seeds/quantiles.
\end{enumerate}

\section{Model components and notation}

\subsection{Time series and splits}
Let $y_{1:T}$ denote the observed time series. We use a time-ordered split:
\[
\{1,\dots,T\} = \mathcal{T}_{\text{train}} \cup \mathcal{T}_{\text{verify}} \cup \mathcal{T}_{\text{test}},
\quad \max(\mathcal{T}_{\text{train}}) < \min(\mathcal{T}_{\text{verify}}) < \min(\mathcal{T}_{\text{test}}).
\]
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item \textbf{Training block} $\mathcal{T}_{\text{train}}$: used for fitting each quantile model under a candidate DESN.
\item \textbf{Verification block} $\mathcal{T}_{\text{verify}}$: used only for model selection scoring and ranking.
\item \textbf{Test/forecast block} $\mathcal{T}_{\text{test}}$: used once for final reporting after selection.
\end{itemize}

\paragraph{Repo-native split controls.}
The pipelines use \texttt{cfg$split} (e.g., \texttt{train\_prop}, \texttt{train\_n}, \texttt{T\_use}, \texttt{use\_last}) and then compute train/forecast splits internally. Model selection uses \texttt{model\_selection$split} (see \texttt{R/qdesn\_model\_selection.R}).

\subsection{Rolling-origin evaluation}
Let $\mathcal{O}$ be the set of forecast origins within $\mathcal{T}_{\text{verify}}$ and $\mathcal{H}$ the horizon set. The repo uses \texttt{forecast\_lattice.qdesn\_fit()} to handle multi-origin forecasting. Weights over $(t_0, h)$ are configured in \texttt{cfg$forecast} (e.g., \texttt{lead\_weight\_power}, \texttt{eval\_leads}).

\subsection{Quantile grid and synthesis}
Let $\mathcal{Q}=\{\tau_1,\dots,\tau_M\}$ be the quantile set (\texttt{p\_vec} in config). Synthesis is handled by
\texttt{exdqlm\_synthesize\_from\_draws()} with isotonic and rearrangement options controlled by \texttt{cfg$synthesis}.

\section{Candidate DESN search space}

\subsection{Primary tuned parameters (repo-native)}
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item $D$ (\texttt{cfg$desn$D}): number of reservoir layers
\item $n$ (\texttt{cfg$desn$n}): units per layer (length $D$)
\item $n\_\tilde{}$ (\texttt{cfg$desn$n\_tilde}): inter-layer projection dims (length $D-1$)
\item $m$ (\texttt{cfg$desn$m}): number of output lags in reservoir input
\item $\alpha$ (\texttt{cfg$desn$alpha}): leakage parameter(s)
\item $\rho$ (\texttt{cfg$desn$rho}): spectral scale(s)
\end{itemize}

\subsection{Fixed DESN parameters (default)}
Hold the following constant during selection unless a study explicitly varies them:
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item activations (\texttt{act\_f}, \texttt{act\_k})
\item sparsity (\texttt{pi\_w}, \texttt{pi\_in})
\item washout (\texttt{washout})
\item readout bias (\texttt{add\_bias})
\item seeds (\texttt{desn$seed}): scalar or per-layer vector; scalar is expanded per layer in \texttt{pipeline\_run.R}
\end{itemize}

\section{Scoring metrics (repo-native)}

\subsection{Primary metric: CRPS for synthesized distribution}
In pipelines and model-selection utilities, CRPS is computed from synthesized draws and summarized in \texttt{metrics\_summary.csv} with \texttt{component = "synth"} and \texttt{score = "crps"} or \texttt{"crps\_synth"}. The selection objective typically uses the forecast/verification window.

\subsection{Calibration diagnostics}
Pipelines compute PIT and coverage-based calibration diagnostics for each $\tau$ and (optionally) synthesized distributions. These are stored in tables/plots and can be used as constraints or penalties. The doc's $\CalRMSE$ can be mapped to coverage error summaries already produced in the diagnostics tables, or added as a planned extension (see \S\ref{sec:planned}).

\section{Selection objective}

\subsection{Option A: constrained optimization (recommended default)}
\[
\min_\theta \; S_{\mathrm{CRPS}}(\theta)
\quad \text{subject to}\quad
\overline{\CalRMSE}(\theta)\le \varepsilon.
\]

\subsection{Option B: weighted sum}
\[
J_\lambda(\theta) = S_{\mathrm{CRPS}}(\theta) + \lambda\,\overline{\CalRMSE}(\theta).
\]
\textbf{Note:} current repo code primarily implements CRPS(synth) selection; explicit calibration penalties are planned.

\section{Handling ESN stochasticity (seeds)}
Reservoir weights are random given seeds. In repo terms, use \texttt{cfg$desn$seed} as a candidate-level seed (scalar or vector). For robustness, evaluate each candidate across a fixed seed list and average the objective. Existing model-selection utilities accept explicit \texttt{seed\_vec} (see \texttt{R/model\_selection\_distribution\_first.R}).

\section{Search procedure: multi-stage, multi-fidelity}
The repo already contains two paths:
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item \textbf{Distribution-first selection:} \texttt{model\_selection\_distribution\_first()} with \texttt{stage = c("coarse","final")} controlling fidelity.
\item \textbf{Config-driven selection:} \texttt{qdesn\_model\_selection()} using a \texttt{model\_selection} block in a spec YAML.
\end{itemize}
This document's multi-fidelity stages map naturally to these functions.

\section{Implementation blueprint (aligned to repo)}
\label{sec:planned}

\subsection{Existing components}
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item \texttt{R/qdesn\_model\_selection.R}: end-to-end selection driver (train/verify/test)
\item \texttt{R/model\_selection\_distribution\_first.R}: distribution-first CRPS-based screening

\subsection{Planned components (doc-only; no implementation yet)}
The following are not yet formalized as public APIs and should be treated as planned additions:
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item \texttt{evaluate\_candidate()} \emph{(planned)}: should live in \texttt{R/qdesn\_model\_selection.R} next to \texttt{.qdesn\_ms\_*} helpers.
\item Calibration penalty/constraint \emph{(planned)}: best added to \texttt{.qdesn\_ms\_read\_metrics()} or a new helper in \texttt{R/qdesn\_model\_selection.R}.
\item Candidate caching \emph{(planned)}: likely in \texttt{run\_esn\_pipeline\_from\_cfg()} (if added) or a new helper in \texttt{R/qdesn\_model\_selection.R}.
\end{itemize}

\section{Output conventions (repo-native)}
Pipeline outputs are written under:
\[
\texttt{results/<suite>/<dataset>/runs/<run\_id>/}
\]
Key artifacts include:
\begin{itemize}[itemsep=2pt, topsep=4pt]
\item \texttt{manifest/cfg\_effective.yaml} and \texttt{run\_manifest.json}
\item \texttt{tables/metrics\_summary.csv} (used for CRPS selection)
\item \texttt{figs/} and \texttt{models/} directories
\item \texttt{logs/main.log}
\end{itemize}

\end{enumerate}
